Animation Frames

60FPS ist das Maß der Dinge

Was sind Animation Frames?

  • Synchronisieren Funktionsaufrufe mit der Bildwiederholrate
    ("animation frame")
  • Angefordert mit window.requestAnimationFrame(…)
  • Maximal 60 Animation Frames pro Sekunde
  • CSS Animationen benutzen die gleichen Frames

Animation Frames werden

aggregiert

und als synchroner Block ausgeführt


function printTimeStamp(timeStamp) {
    document.getElementById('agg-out')
        .textContent += timeStamp + '\n';
}

// Those all fire at the same time
requestAnimationFrame(printTimeStamp);
for (let i = 0; i < 1000; i++) {}
requestAnimationFrame(printTimeStamp);
requestAnimationFrame(printTimeStamp);
for (let i = 0; i < 1000; i++) {}
requestAnimationFrame(printTimeStamp);

Live-Editing

Worum geht's?

  • Live-Editor und Ausführung auf der Kommandozeile
  • Im Editor können Inhalte editiert werden
  • Im Präsentationsfenster werden sie sofort aktualisiert
  • Editierte Inhalte werden sofort gespeichert
  • Beliebige Kommandozeilenbefehle können ausgeführt werden

How To - Editor

Für jedes File, das editiert werden soll:

  • Im Element <select> das Attribut "data-base-path" auf den Basispfad ändern
  • Ein Eintrag <option> pro editierbarem File:
    • Attribut "data-file": Dateiname relativ zum Basispfad
    • Attribut "data-language": Verwendete Sprache
    • Attribut "data-execute": Befehl für die Ausführung

How To - Präsentation

Anzeige des Codes einer Datei:

  • HTML-Elemente <pre><code> anlegen:
  • Dem Code-Tag eine ID geben
  • Attribut "data-file": Dateiname (inkl. Basispfad)
  • Attribut "data-language": Verwendete Sprache

Anzeige des Outputs der Ausführung:

  • Den Block mit Klasse "output-section" kopieren
  • Attribut "data-file" des <code>-Tage:
    Dateiname (inkl Basispfad)
